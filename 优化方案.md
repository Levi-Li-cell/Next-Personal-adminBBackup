# 后台管理系统优化方案

## 1. 数据库结构优化

### 1.1 现有表结构分析

- **user表**：包含用户基本信息和角色字段
- **blog表**：包含博客文章信息
- **project表**：包含项目信息

### 1.2 数据库优化方案

1. **添加图片链接数组字段**：
   - 在blog表中添加 `imageLinks` 字段（JSON类型，存储vercel返回的图片链接数组）
   - 在project表中添加 `imageLinks` 字段（JSON类型，存储vercel返回的图片链接数组）
   - 在user表中添加 `imageLinks` 字段（JSON类型，存储用户相关的图片链接数组）

2. **用户权限管理优化**：
   - 保留现有的 `role` 字段（member、admin等）
   - 可考虑添加 `permissions` 字段（JSON类型），存储更细粒度的权限控制

### 1.3 数据库迁移计划

1. 使用 Drizzle ORM 生成数据库迁移文件
2. 执行迁移命令将更改应用到 Supabase 数据库
3. 验证数据库结构是否正确

## 2. 侧边栏管理功能优化

### 2.1 现有侧边栏结构

- 仪表盘

- 项目管理
- 用户管理

### 2.2 侧边栏优化方案

1. **保留的功能**：
   - 仪表盘
   - 博客管理
   - 项目管理
   - 用户管理
   - 通知中心

2. **优化内容**：

   - 实现侧边栏的响应式设计，在移动设备上可折叠

## 3. 富文本编辑器集成

### 3.1 选择的富文本编辑器

- **TinyMCE**：功能丰富的富文本编辑器

### 3.2 集成计划

1. **安装依赖**：
   - 安装 `tinymce` 包
   - 安装 `@tinymce/tinymce-react` 包（React 组件）

2. **配置编辑器**：
   - 在博客和项目的创建/编辑页面集成 TinyMCE
   - 配置编辑器的工具栏、插件和样式
   - 实现图片上传功能，将图片上传到 Vercel Blob 并存储链接

3. **数据处理**：
   - 将编辑器内容保存到数据库的 `content` 字段
   - 将图片链接保存到新增的 `imageLinks` 字段

## 4. 前台渲染功能

### 4.1 博客前台渲染

1. **博客列表页面**：
   - 创建或优化 `/blog` 页面，展示博客文章列表
   - 实现文章分类、标签筛选功能

2. **博客详情页面**：

   - 支持富文本内容渲染
   - 显示文章的作者、发布时间、分类和标签
   - 实现相关文章推荐功能

### 4.2 项目前台渲染

### 4.3 数据获取与处理

1. **API 路由**：

   - 实现数据分页、筛选和排序功能

2. **响应式设计**：

### 5.2 顶部导航栏优化
